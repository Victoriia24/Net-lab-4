1. Теоретический раздел.

1.1. Часть 1. Получение доступа к Интернет из локальной сети и ознакомление с сервисами Интернет.

Internet является общедоступной сетью, открытой для любого пользователя. Доступ в Internet через постоянное сетевое соединение или коммутируемую линию предоставляется провайдером услуг Internet (ISP - Internet Service Provider). ISP предлагают высокоскоростные соединения по всему миру. Стоит это в несколько раз дешевле, чем содержание частной сети типа WAN (Wide Area Network). Такими провайдерами могут быть как небольшие фирмы, осуществляющие локальные операции, так и крупные национальные организации, предлагающие широкий спектр услуг и возможностей подключения.

Основными сервисами Интернет считаются WEB (HTTP протокол), FTP, E-Mail (SMTP), USENet (NNTP), ТЕLNЕТ. Для доступа к сервисам требуется осуществить подключенгие вашего браузера к основному серверу, осуществляющему трансляцию всех ваших запросов во внешнюю сеть. В институтской сети таким сервером является сервер - WiGate с клиентской частью gatekeeper.

1.2. Часть 2. Ознакомление с работой на поисковых серверах.

Поиск информации в Интернет осуществляется с помощью браузеров, при условии, что ваш компьютер подключен по TCP/IP к Internet. Каждый ресурс в Internet имеет свой уникальный адрес, получит доступ к этому ресурсу можно по его URL.

Информащия в Iпtеrnеt носит распределенный характер и находится на сотнях тысячах компьютеров, подключенных к сети. Как известно, в INTERNET нет единого, центрального архива. Поэтому поиск информации в INTERNET осуществляется с помощью специальных поисковых служб (движков) (searсh еnginе). Они представляют собой программы, выполняющие поиск в постоянно обновляемых базах данных, содержащих URL, ключевые слова и краткие описания для WеЬ-страниц.

Поисковые машины не дадут точного ответа на Ваш вопрос, а лишь укажут список адресов WWW-серверов, на которых вы были обнаружены ссылки. В случае необходимости провести сложный поиск стоит использовать возможность применить логические операторы при построении запроса. Практически все поисковые машины позволяют организовать поиск с использованием основных операторов булевой логики (АND, ОR и т.д.).

Весь Интернет работает в пространстве IP адресов, но для удобства их интерпретации и запоминания используют также символьные адреса. Эти адреса называют – DNS адресами.

База данных DNS имеет структуру дерева, называемого доменным пространством имен, в котором каждый домен (узел дерева) имеет имя и может содержать поддомены. Имя домена идентифицирует его положение в этой базе данных по отношению к родительскому домену, причем точки в имени отделяют части, соответствующие узлам домена. Для обозначения стран используются трехбуквенные и двухбуквенные аббревиатуры, а для различных типов организаций используются следующие аббревиатуры:

  • com - коммерческие организации (например, microsoft.com);
  • edu - образовательные (например, mit.edu);
  • gov - правительственные организации (например, nsf.gov);
  • org - некоммерческие организации (например, fidonet.org);
  • net - организации, поддерживающие сети (например, nsf.net).

Каждый домен DNS администрируется отдельной организацией, которая обычно разбивает свой домен на поддомены и передает функции администрирования этих поддоменов другим организациям. МЕжду DNS-именем и IP адресом всегда есть полное сответствие. Для любого DNS имени есть хотя бы один IP адрес. IP адрес может не иметь DNS-имени. IP адрес может иметь несколько DNS имен.

1.3. Часть 6. Получение информации с FTP серверов

FTP - File Transfer Protocol - это протокол передачи данных. Он предназначен для передачи файлов с удаленного компьютера на локальный. Этоп протокол позволяет получит доступ к сервису FTP, поддерживаемому соответствующим сервером. Для соединения с этим сервисом требуется ввести пароль, обычно это слово anonymous - это имя анонимного пользователя.

URL для ftp выглядит так:
ftp://<user>:<password>@<host>:<port>/<url-path>,
где user - имя пользователя, password - его пароль, host - доменное имя или IP адрес сервера, url-path - путь к файлу. Для анонимного ftp в url сделан упрощенный синтаксис: ftp://<host>/<url-path>, то есть при отсутствии имени автоматически будет вставлено anonymous. Порт также обычно не указывается, а используется стандартный 21.

Примеры адресов ftp в форме url:
ftp://ftp.cdrom.com/pub/music/songs/1996
ftp://ds.internic.net/rfc/rfc1738.txt
В качестве host можно указывать и IP адрес. (118.24.64.24).

Далее вы просто попадаете в файловую систему ( каталог), находящуюся на удаленной машине. Файлы распределены по каталогам - директориям или папкам.Так как традиционной операционной системой для многопользовательских систем является UNIX, то везде Вам придется использовать имена каталогов в стиле UNIX. Общий вид пути в файлу выглядит так:
/имя_каталога/имя_подкаталога/имя_файла,
где имя_подкаталога может повторяться сколь угодно раз или его может не быть вовсе. Наверное, Вы уже заметили, что имена каталогов разделяются знаком '/'.Учтите, что маленькие и большие буквы различаются. Операционную систему удаленного компьютера можно узнать, введя system. Какие порты бывают?

Порты являются виртуальными путями, по которым информация передается от компьютера к компьютеру. Всего есть 65536 портов на выбор.

  • Порты 0 до 1023 - Самые известные номера портов. Наиболее популярные службы работающие на портах: база данных MS SQL (1433), почтовые услуги POP3 (110), IMAP (143), SMTP (25), веб-сервисы HTML (80).
  • Порты от 1024 до 49151 - зарезервированные порты; это означает, что они могут быть зарезервированные для конкретных протоколов программного обеспечения.
  • Порты 49152 по 65536 - динамические или частные порты; это означает, что они могут быть использованы кем угодно.

2.4.2. Как повысить эффективность поиска документов в INTERNET.

Итак, как же правильно «общаться» с поисковой системой Google?

Во-первых, чем проще – тем лучше. Если хотим узнать, к примеру, что такое веб-квест, то так и пропишем в строке поиска: веб-квест.

Google поймет наш запрос, даже если написать его без заглавных букв, с орфографическими ошибками или опечатками. Это особенно ценно для новичков, не до конца освоивших клавиатуру.

Далее, необходимо точнее формулировать запрос. К примеру, лучше прописать «IPhone», а не «мне подарили IPhone». В первом случае в результатах поиска отобразится информация про гаджет, а во втором – рассказы пользователей о том, как им подарили IPhone.

Если необходимо найти точную фразу, она берётся в кавычки.

Поставьте плюс перед теми словами в запросе, которые имеют бо́льший вес для Вас.

Если из результатов поиска нужно исключить лишнее, ставим перед ним знак минуса. Например, чтобы найти в Сети информацию о Наполеоне-императоре (а не о торте «Наполеон»), нужно прописать в строке поиска: Наполеон -торт.

Если хотите поискать имеющиеся ссылки по конкретному сайту, то введите название сайта со словом site. Например: site:cpto2.com.ua. Увидите, что поисковик даёт на этот сайт несколько тысяч ссылок.

Если необходимо найти сайты со схожей тематикой, то вводим название сайта после слова related. Например: related: cpto2.com.ua.

Можно указывать тип документа, что удобно при поиске статей, презентаций, учебников: filetype:pdf.

Если хотим узнать значение какого-либо слова, то в строке поиска вводим его после оператора «define:».

Чтобы узнать курс обмена валют, достаточно ввести запрос такого типа: 100 евро в гривны.

Google также поможет решить математический пример, даже самый сложный; просто введите его в строку поиска, поставив в конец знак равенства (=).

Читаем новости по интересующим нас темам. Если нажать на кнопку «Новости», откроется информация о последних событиях. Можно искать информацию и по заданному кругу вопросов. Например, введите запрос «крикет» в разделе «новости» и увидите не правила игры, а результаты последних турниров.

Вы можете найти свой дом или другой объект на карте. Нажимаем на кнопку «Карты», в строку поиска вводим интересующий нас адрес. Перед нами откроется карта. Чтобы увидеть панорамный снимок, перетащим желтого человечка на карту. Также на карте можно построить свой маршрут.
